📁 Dosya Özellikleri Rehberi

Proje dosyalarının detaylı açıklamaları ve kullanım kılavuzu

📂 Proje Dosya Yapısı

📁 ANA KLASÖRLER:
├── src/: Kaynak kodların bulunduğu ana klasör
├── public/: Statik dosyalar (resimler, ikonlar, PDF'ler)
├── backups/: Yedek dosyalar
└── components/: React bileşenleri

📁 SRC KLASÖRÜ İÇERİĞİ:
├── app/: Next.js 13+ App Router dosyaları
├── components/: Yeniden kullanılabilir React bileşenleri
├── lib/: Yardımcı fonksiyonlar ve kütüphaneler
├── types/: TypeScript tip tanımlamaları
└── utils/: Genel yardımcı fonksiyonlar

⚙️ Konfigürasyon Dosyaları

package.json.json
Ne işe yarar: Proje bağımlılıklarını ve script'lerini tanımlar
İçerik: npm paketleri, versiyonlar, çalıştırma komutları
Projenin temel yapılandırma dosyası
package-lock.json.json
Ne işe yarar: Tam paket versiyonlarını kilitleyerek tutarlılık sağlar
İçerik: Her paketin kesin versiyonu ve bağımlılıkları
Farklı ortamlarda aynı paket versiyonlarını kullanır
tsconfig.json.json
Ne işe yarar: TypeScript derleyici ayarlarını yapılandırır
İçerik: Derleme seçenekleri, dosya yolları, tip kontrolleri
TypeScript kodunun nasıl derleneceğini belirler
tailwind.config.ts.ts
Ne işe yarar: Tailwind CSS framework'ünün ayarlarını yapılandırır
İçerik: Renk paletleri, boyutlar, özel sınıflar
CSS stillerinin nasıl oluşturulacağını belirler
next.config.ts.ts
Ne işe yarar: Next.js framework'ünün yapılandırma ayarları
İçerik: Build ayarları, optimizasyonlar, plugin'ler
Next.js uygulamasının nasıl çalışacağını belirler
eslint.config.mjs.mjs
Ne işe yarar: ESLint kod kalitesi kontrol ayarları
İçerik: Kod standartları, kurallar, kural dışı bırakmalar
Kod kalitesini ve tutarlılığını sağlar

📄 Dosya Türleri ve Uzantıları

TypeScript Dosyaları.ts/.tsx
Ne işe yarar: Tip güvenli JavaScript geliştirme
Kullanım: React bileşenleri, API fonksiyonları, yardımcı fonksiyonlar
Button.tsx, api.ts, utils.ts
JavaScript Dosyaları.js/.mjs
Ne işe yarar: Dinamik web uygulamaları geliştirme
Kullanım: Konfigürasyon dosyaları, eski format dosyalar
postcss.config.js, eslint.config.mjs
CSS Dosyaları.css
Ne işe yarar: Web sayfalarının görsel stillerini tanımlama
Kullanım: Özel stiller, global CSS, animasyonlar
globals.css, components.css
Markdown Dosyaları.md
Ne işe yarar: Formatlanmış dokümantasyon ve açıklamalar
Kullanım: README dosyaları, proje dokümantasyonu
README.md, CHANGELOG.md
Metin Dosyaları.txt
Ne işe yarar: Basit metin dokümantasyonu
Kullanım: Hızlı notlar, basit açıklamalar
npm-komutlari.txt, paket-listesi.txt
Resim Dosyaları.png/.jpg/.svg
Ne işe yarar: Web sitesi görselleri ve ikonları
Kullanım: Logo, fotoğraflar, ikonlar, arka plan görselleri
eren-photo.png, globe.svg, favicon.ico
PDF Dosyaları.pdf
Ne işe yarar: Belgeler ve sertifikalar
Kullanım: CV, sertifikalar, proje belgeleri
ErenLebensL.pdf, ergon.pdf

📂 Dosya Organizasyonu Prensipleri

📂 Klasör Yapısı:

  • Mantıksal gruplandırma: Benzer dosyalar aynı klasörde
  • Hiyerarşik düzen: Ana klasörler alt klasörlere ayrılır
  • Açıklayıcı isimler: Klasör adları içeriği yansıtır

📝 Dosya İsimlendirme:

  • camelCase: JavaScript/TypeScript dosyaları
  • kebab-case: CSS dosyaları ve klasörler
  • PascalCase: React bileşen dosyaları
  • lowercase: Yapılandırma dosyaları

🔧 İçerik Organizasyonu:

  • Tek sorumluluk: Her dosya tek bir işlevi yerine getirir
  • Modüler yapı: Bileşenler yeniden kullanılabilir
  • Bağımlılık yönetimi: İçe aktarmalar düzenli

🔒 Dosya Güvenliği ve Yedekleme

🔒 Güvenlik:

  • .gitignore: Git'e dahil edilmeyecek dosyalar
  • Environment variables: Hassas bilgiler ayrı dosyalarda
  • Backup klasörü: Önemli dosyaların yedekleri

💾 Yedekleme:

  • backups/ klasörü: Manuel yedekler
  • Git history: Versiyon kontrolü ile otomatik yedekleme
  • Cloud storage: Uzak sunucu yedekleri

📊 Performans:

  • Resim sıkıştırma: PNG, JPG optimizasyonu
  • Code splitting: Büyük dosyaların bölünmesi
  • Tree shaking: Kullanılmayan kodların çıkarılması
  • Minification: Üretim dosyalarının küçültülmesi

⚠️ Önemli Notlar

  • package-lock.json dosyasını manuel olarak düzenlemeyin
  • .env dosyalarını Git'e commit etmeyin
  • node_modules klasörünü manuel olarak silmeyin
  • Build dosyalarını kaynak kodla karıştırmayın

💡 İpuçları:

  • Dosya isimlerini açıklayıcı yapın
  • Klasör yapısını mantıklı organize edin
  • Düzenli yedekleme yapın
  • Dosya boyutlarını kontrol edin
  • Güvenlik güncellemelerini takip edin

🔄 Güncelleme Önerileri:

  • Haftalık: Kullanılmayan dosyaları temizleyin
  • Aylık: Güvenlik güncellemelerini kontrol edin
  • Üç aylık: Performans optimizasyonu yapın
  • Yıllık: Büyük refactoring planlayın
