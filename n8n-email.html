📧 n8n Email Entegrasyonu

Sıfırdan Tam Kurulum Rehberi - Render + GitHub + n8n

🚀 1. Render Hesap Oluşturma

1.1 Render.com'a Kayıt

  • render.com adresine gidin
  • "Get Started" butonuna tıklayın
  • GitHub hesabınızla giriş yapın
  • Email adresinizi doğrulayın

1.2 Render Dashboard

  • Dashboard'a giriş yapın
  • Sol menüde "New" butonunu görün
  • Ücretsiz plan seçin

📁 2. GitHub'da n8n Dosyaları Oluşturma

2.1 Yeni Repository

  • GitHub.com'a gidin
  • "New repository" butonuna tıklayın
  • Repository adı: "n8n-render"
  • Public seçin
  • "Create repository" tıklayın

2.2 package.json Dosyası

Repository'de "Add file" → "Create new file" → "package.json":

{ "name": "n8n-render", "version": "1.0.0", "description": "n8n for Render deployment", "main": "index.js", "scripts": { "start": "n8n start" }, "dependencies": { "n8n": "^1.0.0" }, "engines": { "node": ">=18.0.0" } }

2.3 railway.json Dosyası

"Add file" → "Create new file" → "railway.json":

{ "$schema": "https://railway.app/railway.schema.json", "build": { "builder": "NIXPACKS" }, "deploy": { "startCommand": "n8n start", "healthcheckPath": "/", "healthcheckTimeout": 100, "restartPolicyType": "ON_FAILURE", "restartPolicyMaxRetries": 10 } }

2.4 README.md Dosyası

"Add file" → "Create new file" → "README.md":

# n8n Render Deployment n8n automation platform deployed on Render. ## Environment Variables - N8N_BASIC_AUTH_ACTIVE=true - N8N_BASIC_AUTH_USER=admin - N8N_BASIC_AUTH_PASSWORD=your_password - N8N_HOST=0.0.0.0 - N8N_PORT=3000 - N8N_PROTOCOL=https ## Usage 1. Deploy to Render 2. Set environment variables 3. Access n8n dashboard

2.5 Dosyaları Commit

  • Her dosyayı oluşturduktan sonra "Commit new file" tıklayın
  • 3 dosya da repository'de görünmeli
  • Repository URL'ini kopyalayın

⚙️ 3. Render'da n8n Deploy

3.1 New Web Service

  • Render Dashboard'da "New" → "Web Service"
  • "Connect a repository" → GitHub'ı seçin
  • n8n-render repository'sini seçin
  • "Connect" tıklayın

3.2 Service Ayarları

  • Name: "n8n-render"
  • Region: Frankfurt (EU Central)
  • Branch: main
  • Root Directory: (boş bırakın)
  • Runtime: Node
  • Build Command: npm install
  • Start Command: npm start

3.3 Environment Variables

"Environment" sekmesinde şu değişkenleri ekleyin:

N8N_BASIC_AUTH_ACTIVE=true N8N_BASIC_AUTH_USER=admin N8N_BASIC_AUTH_PASSWORD=your_secure_password N8N_HOST=0.0.0.0 N8N_PORT=3000 N8N_PROTOCOL=https
⚠️ N8N_BASIC_AUTH_PASSWORD için güçlü bir şifre kullanın!

3.4 Deploy

  • "Create Web Service" butonuna tıklayın
  • Deploy işlemini bekleyin (5-10 dakika)
  • "Live" durumunu görün
  • URL'i kopyalayın (örn: https://n8n-render-pkzu.onrender.com)

🔐 4. n8n'e Giriş

4.1 n8n Dashboard

  • Render'dan aldığınız URL'e gidin
  • n8n login sayfası açılacak
  • Username: admin
  • Password: N8N_BASIC_AUTH_PASSWORD'ta yazdığınız şifre
  • "Sign In" tıklayın

4.2 n8n Arayüzü

  • Dashboard'a erişin
  • Sol menüde "Workflows" görün
  • Üst menüde "Editor", "Executions" sekmeleri var
  • Sağ üstte "Create Workflow" butonu var

📧 5. Gmail SMTP Ayarları

5.1 Gmail App Password

  • Google Hesabınıza gidin
  • Güvenlik → 2 Adımlı Doğrulama açın
  • Uygulama Şifreleri → Diğer
  • İsim: "n8n"
  • 16 haneli şifreyi kopyalayın

5.2 n8n'de SMTP Credential

  • n8n'de sol menüden "Credentials" tıklayın
  • "Add Credential" → "SMTP"
  • Aşağıdaki bilgileri girin:
User: eren.yigit.aydin@gmail.com Password: [Gmail App Password - 16 haneli] Host: smtp.gmail.com Port: 465 SSL/TLS: Enabled
✅ "Connection tested successfully" mesajını görmelisiniz

⚙️ 6. Email Workflow Oluşturma

6.1 Yeni Workflow

  • "Create Workflow" butonuna tıklayın
  • Workflow'a isim verin: "Contact Form Email"
  • Editor'a gidin

6.2 Webhook Node'u

  • "Webhook" node'u ekleyin
  • HTTP Method: POST
  • Path: contact-form
  • Authentication: None
  • Respond: "Immediately" (ÖNEMLİ!)
⚠️ "Using 'Respond to Webhook' Node" seçmeyin!

6.3 Send Email Node'u

  • "Send Email" node'u ekleyin
  • Credential: SMTP account (az önce oluşturduğunuz)
  • Operation: Send
  • From Email: eren.yigit.aydin@gmail.com
  • To Email: eren.yigit.aydin@gmail.com
  • Subject: Neuen Portfolio Email
  • Email Format: HTML

6.4 Email Template (KRİTİK!)

İsim: {{$json.body.name}}<br>
Email: {{$json.body.email}}<br>
Mesaj: {{$json.body.message}}
⚠️ {{$json.name}} değil, {{$json.body.name}} kullanın!
Veri "body" objesi içinde geliyor.
ÖNEMLİ:
etiketleri olmadan email tek satırda gelir!

6.5 Node'ları Bağlama

  • Webhook node'undan Send Email node'una ok çekin
  • Bağlantı: [Webhook] → [Send Email]

6.6 Workflow Aktifleştirme

  • Sağ üst köşedeki "Toggle" butonuna tıklayın
  • Workflow yeşil olmalı (Active)

6.7 Webhook URL'ini Alma

  • Webhook node'una tıklayın
  • "Production URL" butonuna tıklayın
  • URL'i kopyalayın
https://n8n-render-pkzu.onrender.com/webhook/contact-form

🔗 7. Next.js API Route Güncelleme

7.1 API Route Dosyası

src/app/api/contact/route.ts dosyasını açın ve şu şekilde güncelleyin:

import { NextRequest, NextResponse } from 'next/server'; export async function POST(request: NextRequest) { try { const data = await request.json(); console.log('API route received data:', data); // n8n webhook'a istek gönder const response = await fetch('https://n8n-render-pkzu.onrender.com/webhook/contact-form', { method: 'POST', headers: { 'Content-Type': 'application/json', }, body: JSON.stringify(data), }); console.log('n8n response status:', response.status); console.log('n8n response ok:', response.ok); if (response.ok) { const responseData = await response.text(); console.log('n8n response data:', responseData); return NextResponse.json({ success: true }); } else { const errorText = await response.text(); console.error('n8n webhook error:', response.status, response.statusText, errorText); return NextResponse.json( { error: 'Webhook error', details: errorText }, { status: 500 } ); } } catch (error) { console.error('API route error:', error); return NextResponse.json( { error: 'Internal server error', details: error instanceof Error ? error.message : 'Unknown error' }, { status: 500 } ); } }
⚠️ URL'deki "n8n-render-pkzu" kısmını kendi Render URL'inizle değiştirin!
